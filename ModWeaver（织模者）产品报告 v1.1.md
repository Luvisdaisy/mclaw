## 一、产品概述

### 1.1 产品定位

ModWeaver是一款**开源免费的Minecraft模组整合包智能构建与局域网协作工具**。通过AI驱动的智能体技术，让玩家用自然语言描述需求即可自动完成模组选择、依赖解析、冲突检测、环境配置与服务器部署，彻底消除Minecraft模组游玩的技术门槛。

**一句话价值主张：** 从想法到联机，五分钟开启属于你的模组世界。

### 1.2 开源承诺

ModWeaver采用MIT许可证，永久免费开源。我们相信模组社区的精神在于分享与协作，这款工具也将回馈社区，允许任何人自由使用、修改和分发。

### 1.3 目标用户

| 用户画像           | 核心痛点                                                     | 使用场景                                         |
| ------------------ | ------------------------------------------------------------ | ------------------------------------------------ |
| **模组新人玩家**   | 不知道选什么模组、看不懂依赖关系、安装过程繁琐、崩溃无从下手 | 想要特定主题体验（魔法/科技/冒险），一键生成即玩 |
| **小团体玩家**     | 四人联机需要统一版本、配置服务器困难、模组同步痛苦           | 朋友间快速搭建统一整合包，局域网直接联机         |
| **整合包体验者**   | 想尝试不同组合，但手动测试耗时巨大                           | 快速实验模组搭配，自动检测可行性                 |
| **轻度整合包作者** | 想分享自用包给朋友，但打包分发复杂                           | 生成一键分享包，接收者自动配置环境               |
| **开源贡献者**     | 希望参与改进Minecraft工具链                                  | 贡献代码、模组知识库、测试用例                   |

### 1.4 产品愿景

让每个人都能拥有专属的Minecraft整合包，让联机像加入局域网游戏一样简单。打造Minecraft模组社区最值得信赖的开源工具。

## 二、核心价值主张

### 2.1 用户价值曲线

```
传统流程：
  选模组 → 查依赖 → 找前置 → 配Java → 调Config → 崩溃 → 二分法排查 → 重复N次 → 终于能玩
  ⏱ 耗时：数小时到数天

ModWeaver流程：
  描述需求 → AI推荐组合 → 一键构建 → 沙箱验证 → 启动即玩
  ⏱ 耗时：5-15分钟
```

### 3.2 核心功能详细说明

#### 功能1：智能整合包构建引擎

**1.1 自然语言需求解析**
- **输入方式**：文本对话、标签选择、模组收藏夹
- **解析能力**：提取玩法类型、核心模组、性能要求、光影偏好
- **示例**：
  - 输入："我想要一个以Create机械动力为核心的科技空岛包，要轻量优化，能上光影，四个人玩"
  - 解析结果：`{核心模组: [Create], 玩法: [Skyblock], 标签: [Tech, Performance, Shader], 玩家数: 4}`

**1.2 模组智能检索与推荐**
- **数据源**：CurseForge API + Modrinth API + 本地缓存索引
- **推荐策略**：
  - 核心匹配：基于语义相似度检索相关模组
  - 生态补充：自动推荐与核心模组有联动的模组
  - 优化补全：自动推荐性能优化模组（Sodium/Lithium等）
  - 光影匹配：自动推荐兼容的光影包和前置

**1.3 依赖树自动解析**
- 递归解析所有前置依赖
- 自动排除纯客户端/服务端模组
- 检测可选依赖，询问用户是否安装增强功能
- 生成完整依赖图可视化

**1.4 冲突预检与知识图谱**
- **已知冲突数据库**：社区维护的模组兼容性矩阵
- **检测维度**：
  - ID冲突（方块/物品/维度/生物群系）
  - Mixin注入冲突
  - API版本不兼容
  - 世界生成重叠
  - 合成表覆盖
- **冲突等级**：
  - 🔴 致命：必然导致崩溃
  - 🟡 警告：可能导致功能异常
  - 🔵 提示：同类型模组功能重叠

**1.5 模组组合快照保存**
- 保存每次构建的组合为快照
- 支持版本回溯与对比
- 成功运行10分钟自动标记为"已验证组合"

---

#### 功能2：Java运行时全自动管理

**2.1 多版本自动检测**
- 启动时扫描系统所有Java安装
- 识别版本号、发行版、架构信息
- 优先级：手动指定 > 系统安装 > 内置运行时

**2.2 兼容性智能匹配**
- MC版本与Java版本映射表

| Minecraft版本 | 推荐Java版本 | 备选方案            |
| ------------- | ------------ | ------------------- |
| 1.7.10及以下  | Java 8       | -                   |
| 1.8 - 1.11.2  | Java 8       | -                   |
| 1.12 - 1.16.5 | Java 8       | Java 11（部分可用） |
| 1.17 - 1.17.1 | Java 16      | Java 17             |
| 1.18 - 1.20.4 | Java 17      | Java 21             |
| 1.20.5+       | Java 21      | -                   |

- 特殊规则：Forge版本特定要求、GraalVM优化场景

**2.3 手动JAVA_HOME指定**
- 图形化路径选择器
- 实时显示选中Java的版本、架构、发行版
- 兼容性验证结果即时反馈
- 支持为不同实例指定不同Java

**2.4 实例级隔离配置**
- 每个实例保存独立启动配置（launch.json）
- 启动时硬编码Java路径，不依赖环境变量
- 自定义JVM参数（内存、GC策略）

---

#### 功能3：模组加载器管理

**3.1 支持的加载器**
- Forge（1.7.10 - 1.20.1）
- NeoForge（1.20.1+，优先推荐）
- Fabric（1.14+）
- Quilt（1.18+）

**3.2 版本自动匹配**
- 用户选择MC版本后自动筛选可用加载器版本
- 标注推荐版本（最新稳定版/LTS版）

**3.3 加载器智能推荐**
- 分析选中的模组列表
- 如果全部为Fabric模组 → 推荐Fabric
- 如果包含Forge专属模组 → 推荐Forge/NeoForge
- 混合情况 → 查找替代品，提供切换建议

---

#### 功能4：沙箱验证系统

**4.1 虚拟环境构建**
- 在临时目录创建隔离的MC实例
- 注入诊断用Agent模组
- 自动配置最小化运行环境

**4.2 自动化启动测试**
- 启动游戏进程
- Agent模组通过WebSocket报告状态：
  - 主菜单加载完成
  - 世界创建成功
  - 玩家生成完毕
  - 区块加载正常
  - TPS稳定
- 超时策略：超过一定时间未进入下一阶段则判定失败

**4.3 崩溃日志智能诊断**
- 自动抓取crash-reports和latest.log
- 解析堆栈跟踪，识别崩溃类型
- 匹配已知问题数据库
- 区分：模组冲突/前置缺失/Java版本错误/内存不足

**4.4 一键修复建议**
- 提供分级的修复方案：
  - 🟢 自动修复：自动下载兼容版本/补丁模组/修改Config
  - 🟡 建议方案：提供选项供用户确认
  - 🔴 需要决策：二选一或寻找平替

---

#### 功能5：局域网联机系统

**5.1 服务端自动部署**
- 根据客户端整合包自动生成服务端
- 自动剔除纯客户端模组
- 自动替换为服务端兼容版本
- 生成server.properties默认配置（在线模式=关，适合局域网）
- 自动同意EULA

**5.2 mDNS服务发现**
- 服务端启动时广播服务
- 广播信息：服务器名称、MC版本、整合包名、当前玩家数
- 客户端自动发现并显示在局域网列表
- 支持手动输入IP:端口加入（兼容模式）

**5.3 防火墙自动配置**
- Windows：netsh advfirewall添加规则
- macOS：socketfilterfw（通常无需额外配置）
- Linux：检测ufw/firewalld并添加规则
- 服务器关闭时提示用户是否保留规则

**5.4 客户端增量同步**
- 房主生成"房主包"和"精简客户端包"
- 客户端包不含服务端组件和重复的Java运行时
- 房主更新整合包后生成差异包
- 客户端自动拉取增量更新

**5.5 服务器控制面板**
- WebSocket实时推送服务器状态
- 在线玩家列表
- TPS/内存占用仪表盘
- 内置控制台（发送命令）
- 一键备份/回档
- 白名单管理

---

#### 功能6：实例管理中心

**6.1 多实例并行管理**
- 同时安装多个整合包
- 实例间完全隔离（独立mods/config/saves/Java）
- 支持克隆、导出、删除

**6.2 模组配置在线编辑**
- 内置文本编辑器修改Config文件
- 语法高亮（TOML/YAML/JSON）
- 提供常见配置项的中文解释

**6.3 存档备份与恢复**
- 手动备份：一键备份当前存档
- 自动备份：服务器关闭时触发
- 备份策略：增量备份+硬链接去重
- 备份管理：查看、恢复、删除历史备份

**6.4 资源占用监控**
- 磁盘占用（每个实例大小）
- 运行时内存/CPU使用
- 清理缓存和临时文件

## 六、技术架构

### 6.1 技术选型原则

基于团队技术栈（Java/Python），采用以下选型：

- **桌面端优先**：用户本地运行，保护隐私，离线可用
- **Python作为核心后端**：丰富的AI/ML生态，快速开发验证
- **Java用于性能敏感模块**：进程管理、文件监控等需要高性能的场景
- **Electron作为桌面壳**：跨平台UI，与Python后端通过IPC通信

### 6.3 技术栈详细说明

#### 桌面壳层（Electron + React）

| 组件     | 技术                   | 原因                              |
| -------- | ---------------------- | --------------------------------- |
| 桌面框架 | Electron               | 跨平台（Win/Mac/Linux），丰富生态 |
| UI框架   | React + TypeScript     | 组件化开发，类型安全              |
| 状态管理 | Zustand                | 轻量，适合中等复杂度应用          |
| UI组件库 | Ant Design / shadcn/ui | 成熟的中文友好组件库              |
| 图表     | ECharts / Recharts     | 服务器状态可视化                  |

#### Python核心后端

| 组件       | 技术                                                   | 原因                                   |
| ---------- | ------------------------------------------------------ | -------------------------------------- |
| Web框架    | FastAPI                                                | 高性能异步，自动API文档，WebSocket支持 |
| AI编排     | LangChain / LlamaIndex                                 | 构建Agent工作流，RAG检索               |
| LLM        | 本地: Ollama + Qwen/DeepSeek<br>云端: OpenAI API(可选) | 本地优先，隐私保护；云端可选增强       |
| 向量数据库 | ChromaDB                                               | 轻量嵌入式，无需独立部署               |
| 关系数据库 | SQLite + SQLAlchemy                                    | 零配置，数据在用户本地                 |
| 异步下载   | aiohttp + aiofiles                                     | 高性能并发下载                         |
| mDNS       | python-zeroconf                                        | 跨平台服务发现                         |
| 模板引擎   | Jinja2                                                 | 生成启动脚本/配置文件                  |
| 任务队列   | Celery + Redis(可选)                                   | 后台任务处理（可降级为asyncio）        |

#### Java运行时服务（可选组件）

| 组件      | 技术                     | 原因                  |
| --------- | ------------------------ | --------------------- |
| 开发语言  | Java 17+                 | 与MC生态一致          |
| 进程管理  | java.lang.ProcessBuilder | 原生API，无需额外依赖 |
| WebSocket | java.net.http.WebSocket  | JDK内置               |
| 崩溃分析  | ASM / Mixin              | 字节码级诊断          |
| 打包      | jlink (定制JRE)          | 精简运行时            |

### 6.4 为什么这样选型？

#### Python作为核心的优势

1. **AI/ML生态成熟**：LangChain、LlamaIndex、Ollama等库无缝集成
2. **开发效率高**：快速迭代，适合MVP验证
3. **跨平台一致**：无需编译，直接运行
4. **丰富的网络库**：aiohttp、zeroconf等开箱即用

#### Python的挑战与应对

| 挑战               | 应对方案                                         |
| ------------------ | ------------------------------------------------ |
| 性能（高并发下载） | 使用asyncio+aiohttp异步IO，实际瓶颈在网速        |
| 打包分发           | PyInstaller打包为独立可执行文件                  |
| 进程管理           | 使用asyncio.subprocess，对于复杂场景调用Java辅助 |
| 内存占用           | FastAPI本身轻量，LLM使用量化模型                 |

#### 何时使用Java组件

Java运行时服务是可选的增强组件，仅在以下场景使用：
- 需要精确监控Minecraft进程状态
- 需要在JVM层面注入诊断代码（Agent模组）
- 需要解析Minecraft特定的二进制格式

初期MVP可以用纯Python实现核心功能，Java组件在需要时逐步引入。

## 十一、总结

ModWeaver是一个开源免费的Minecraft模组整合包智能构建工具，旨在彻底消除模组游玩的技术门槛。

**技术选择上**：
- 以Python作为核心后端，充分利用其AI/ML生态优势
- Electron + React构建跨平台桌面界面
- Java作为可选辅助组件处理Minecraft特定需求
- 整体架构轻量、可扩展，适合开源社区协作

**MVP核心目标**：让一个完全不懂模组的玩家，能够在15分钟内完成从"我有一个想法"到"和朋友们一起玩"的全过程。

**开源承诺**：MIT许可证，永久免费，欢迎社区贡献。
